## 🔄 更新代碼流程

當您修改了任何檔案（HTML、CSS、JavaScript）後，需要將更新推送到 GitHub 才會在網站上生效。

### 完整更新步驟

1. **打開終端機（Terminal）**
   - 進入專案資料夾：
   ```bash
   cd /Users/huangchonggao/Desktop/pomodoro-timer
   ```

2. **檢查修改狀態**（可選）
   ```bash
   git status
   ```
   - 這會顯示哪些檔案被修改了

3. **提交所有變更**
   ```bash
   git add .
   git commit -m "更新說明"
   ```
   - 將 `"更新說明"` 改成你這次修改的內容，例如：
     - `"新增音效功能"`
     - `"修正計時器錯誤"`
     - `"調整介面配色"`

4. **推送到 GitHub**
   ```bash
   git push
   ```

5. **等待部署完成**
   - 等待 1-3 分鐘
   - GitHub Pages 會自動更新
   - 重新整理網站即可看到更新

### 快速更新指令（一行完成）

如果你很熟悉 Git，可以用一行指令完成：
```bash
git add . && git commit -m "更新內容" && git push
```

## 🎯 功能說明

### 基本操作
- **開始計時**：點擊「▶ 開始」按鈕
- **暫停計時**：點擊「⏸ 暫停」按鈕
- **重置計時**：點擊「↻ 重置」按鈕

### 時長調整
- **快速調整**：使用 ±1 或 ±5 分鐘按鈕
- **範圍限制**：1-120 分鐘
- **注意**：只能在計時器停止時調整

### 模式切換
- **工作模式**：預設 25 分鐘
- **短休息**：預設 5 分鐘
- **長休息**：預設 15 分鐘

### 鍵盤快捷鍵
- **空白鍵（Space）**：開始/暫停計時
- **R 鍵**：重置計時器

### 設定選項
點擊「⚙ 設定」開啟設定面板：
- 自訂各模式時長
- 開啟/關閉聲音提醒
- 開啟/關閉桌面通知
- 自動開始下一個階段

## 📊 統計功能

應用程式會自動記錄：
- **今日統計**：完成次數、工作分鐘、休息次數

## ❓ 常見問題

### Q: 更新代碼後，網站沒有變化？
**A:**
1. 確認 `git push` 成功執行
2. 等待 3-5 分鐘讓 GitHub Pages 完成部署
3. 清除瀏覽器快取：按 `Ctrl/Cmd + Shift + R` 強制重新整理

### Q: 如何確認網站是否已更新？
**A:**
1. 到 GitHub repository：https://github.com/zero6445z-collab/pomodoro-timer
2. 點擊上方的「Actions」標籤
3. 查看最新的部署狀態（綠色勾勾表示成功）

### Q: 可以在離線狀態下使用嗎？
**A:**
第一次訪問後，瀏覽器會快取部分資源，但建議在有網路時使用以確保最佳體驗。

### Q: 統計資料會同步到其他裝置嗎？
**A:**
不會。統計資料儲存在各裝置的瀏覽器中，不同裝置間不會同步。

### Q: 如何重置所有統計資料？
**A:**
點擊統計區塊下方的「重置所有統計」按鈕，或清除瀏覽器 localStorage。
